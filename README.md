# hlk-ld2450-breath-detector

Детектор дыхания человека на базе 24 ГГц радара **HLK-LD2450** и **ESP32**.

Архитектура «тонкий форвардер + тяжёлый анализ на ПК»:

```
HLK-LD2450 ──UART(256000)──> ESP32 ──USB CDC(921600)──> monitor.py (Python)
   (кадры)        парсинг         (raw + fw-BPM)          (band-pass/FFT/апноэ)
```

ESP32 только парсит кадры радара, выбирает стационарную цель в зоне и
транслирует сырые координаты + радиальную дистанцию + лёгкую оценку BPM
(для сравнения). Вся фильтрация, спектральный анализ, оценка SNR/качества и
детекция апноэ выполняются на ПК в `monitor.py`.

## Аппаратная часть
- LD2450: 5V, UART 256000 8N1, выход уровня 3.3V. По умолчанию 10 кадров/с.
- Подключение к ESP32: RX радара <- TX ESP32 (pin 33), TX радара -> RX ESP32 (pin 32).
- Рекомендуемая высота крепления 1.5–2 м, антенна смотрит в зону отслеживания.

## Протокол (USB CDC, текст, одна строка на сообщение)
```
D<dist>,<ac>,<depth>,<lateral>,<visible>,<in_zone>,<stationary>,<ts_ms>,<frame_id>
S<bpm>,<amp>,<qual>,<visible>,<in_zone>,<stationary>,<ts_ms>,<frame_id>
```
- `D` — на каждый кадр радара (минимальная задержка):
  - `dist` — отфильтрованная радиальная дистанция, мм
  - `ac` — детрендированный сигнал дыхания (lp − trend), мм
  - `depth` — радиальная дистанция `r = sqrt(x²+y²)`, мм (ось дыхания)
  - `lateral` — поперечная координата `x`, мм
  - `visible`/`in_zone`/`stationary` — флаги (0/1)
  - `ts_ms` — `millis()` на ESP32
  - `frame_id` — монотонный счётчик кадров
- `S` — сводка раз в ~500 мс: `bpm`, `amp` (пик-пик, мм), `qual` (0..100, fw-оценка).

Префикс `D`/`S` позволяет ПК демультиплексировать поток без неоднозначностей.

## Конвенция координат
- `x` — поперечная (латеральная) ось, мм, со знаком
- `y` — продольная (вперёд от радара) ось, мм
- `depth` = `radial` = `sqrt(x² + y²)` — ось дыхания (грудь движется вдоль r)
- `lateral` = `x` — только для зоны/тепловой карты
- зона детекции: `ZONE_R_MIN(600) ≤ radial ≤ ZONE_R_MAX(2500)` и `|x| ≤ ZONE_SIDE_MAX(500)`

## Сборка и прошивка (ESP32)
Требуется [PlatformIO](https://platformio.org/).
```bash
pio run -t upload          # прошивка
pio device monitor -b 921600
```

## Запуск монитора (ПК)
```bash
pip install -r requirements.txt
python monitor.py COM3 921600      # или просто python monitor.py (автопоиск порта)
```
Backend matplotlib можно переопределить через переменную окружения:
```bash
set MPLBACKEND=QtAgg     # Windows
MPLBACKEND=QtAgg python monitor.py /dev/ttyUSB0
```

### Что делает monitor.py
1. Читает сериал в фоновом потоке (thread-safe буфер).
2. Resample сигнала `ac` на равномерную сетку по `ts_ms` радара.
3. Полосовой фильтр Баттерворта 0.12–0.5 Гц (дыхание ≈ 7–30 вдох/мин).
4. FFT → поиск пика в полосе → BPM = f·60.
5. SNR относительно остальной полосы → качество и порог детекции.
6. Апноэ: отсутствие достоверного пика > 15 с.

## Ограничения
LD2450 — трекер *движущихся* целей; разрешение по дистанции грубое
(`distance_res` ~320 мм в примере datasheet). Амплитуда дыхания (~5–15 мм)
сопоставима с шумом трекера, поэтому ожидайте низкий SNR. Для устойчивой
работы: брать только стационарную цель в зоне, агрессивный полосовой фильтр
и оценку SNR/качества перед выдачей результата.
